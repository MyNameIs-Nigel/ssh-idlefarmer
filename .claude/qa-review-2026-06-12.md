# QA Review — Full-Codebase Audit (2026-06-12)

Whole-system QA pass over the base code (server, game lifecycle, persistence,
sim engine plumbing, TUI wiring) on `beta/gameplay-overhaul`, covering the
five task files still in `.claude/qa/` after the TUI and gameplay overhauls.
Gameplay balance was out of scope per the request; the probability/engine
items below are correctness bugs, not tuning opinions.

**Verification:** `go vet ./...` clean, `gofmt -l` clean, `go build` clean,
`go test ./... -count=1` all green, including 4 new regression tests.

## Bugs found and fixed

1. **Double tick loop** — `tui/game.go Init` batched both `refresh()` (an
   immediate `tickMsg`) and `tickCmd()`, and every `tickMsg` handler schedules
   the next tick. Result: two parallel 1 Hz tick chains per session, forever —
   every connected session advanced/cloned/rendered **twice per second** and
   all "per tick" probabilities ran at double rate. Fixed by seeding a single
   chain.

2. **Online gifts could never arrive** — `rollGiftArrival` computed
   `elapsed*100/interval` in integer percent. With 1–2 s ticks against the
   1200 s online interval that floors to 0, so the roll never fired; gifts
   only ever arrived via offline gaps. Reworked to basis points (1/100 %).
   Regression test: `TestOnlineGiftArrivalRateMatchesInterval`.

3. **Online events fired ~15× too often** — `rollOnlineEvent` had the inverse
   bug: `100/interval` floors to 0 and was then clamped **up** to 1 % per
   tick, giving a ~100 s mean wait instead of the configured 900–1500 s (and
   ~50 s with bug #1 stacked). Same basis-point fix; the chance now also
   scales by `elapsed`. Regression test: `TestOnlineEventRateMatchesInterval`.

4. **Hardier Strains survived rebirth** — documented as run-scoped (coins,
   Market tab), but `Rebirth` reset Tools/Zones/Multipliers and skipped
   `SeedUpgrades`. Now reset; `.claude/gameplay-mechanics.md` updated.
   Regression test: `TestRebirthResetsSeedUpgrades`.

5. **Gift jitter pierced the ceiling** — `giftCoinReward` clamped to the
   ceiling *before* applying ±20 % jitter, so rich runs could be paid up to
   6,000c against a 5,000c ceiling. Clamped after jitter. Regression test:
   `TestGiftCoinRewardNeverExceedsCeiling`.

6. **Shutdown/attach race** — `main.go` runs the save-flush hook *before*
   `srv.Shutdown`, so the SSH listener is still accepting while the manager
   flushes; a player attaching in that window got an actor that was never
   flushed (silent progress loss on redeploy). `Manager` now sets a `closing`
   flag and `Attach` returns `ErrShuttingDown` during the flush. Covered in
   `TestShutdownFlushesAndKicksEveryone`.

7. **Unbounded auto-sow catch-up loop** — when batching couldn't apply
   (unprofitable crop EV ≤ seed cost), the per-cycle loop in `Advance` ran
   `elapsed/grow` iterations with no cap — a 1 s-grow crop offline for months
   means millions of RNG rolls inside the actor while holding the manager
   lock. The loop now breaks at `autoCycleCap`; `PlantedAt` is the loop
   cursor, so later Advances settle the remainder (no progress lost).

## Smaller fixes

- `tui` farm-name input: ASCII-only and no spaces could be typed (sim allows
  24 unicode runes); input also grew unbounded until Enter. Now accepts any
  single printable rune incl. `space`, capped at 24 runes while typing.
- `sim.Plant`: removed a redundant `Critter = ""` after a full struct assign.
- `ev.AutoCoins` accumulation now uses `satAdd` consistently.
- `gofmt -w` across the repo (genuine misalignments in `actor.go`,
  `content.go`, `limits.go`, the `Events` struct, etc.). Note: most of the
  22-file `gofmt -l` output was CRLF noise from `core.autocrlf=true`; only a
  handful had real diffs, and git normalizes the rest away.

## Noted, not changed (judgment calls)

- **`Manager.Attach` holds the global mutex across DB I/O and offline
  catch-up** — serializes all attaches process-wide. Fine at this scale
  (single SQLite conn anyway); revisit only if connect latency matters.
- **Stargazer duration bonus is hardcoded** — `rollOnlineEvent` adds
  `UpgradeLevel("stargazer")*10` for duration because one upgrade can only
  carry one effect. Works, but violates "content is data, not code" and would
  double-count if an `event_duration_pct` upgrade with that id were added.
- **`harvestAllReady`** does one synchronous actor round-trip per plot (max
  ~30). Harmless; a batched intent would be cleaner if plots grow.
- **`kick` callback in `Manager.Attach` is always nil in production** — the
  server relies solely on the `Kicked()` channel. The parameter is dead
  weight outside tests.
- **Critter visit rate** (~2 %/tick per empty plot ≈ one per ~50 s) feels
  high but is cosmetic and self-limiting (one per plot until shooed).
- Overall architecture verdict: the actor/single-writer model, store
  fingerprint-scoping, middleware ordering, sanitize-in/escape-out, and the
  deterministic sim are all genuinely implemented as the overview demands —
  no faked checkmarks found. Test suites are real and behavioral.
